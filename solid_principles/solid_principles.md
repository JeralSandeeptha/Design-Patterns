# Solid Priciples

`SOLID principles` are five design guidelines in object-oriented programming intended to make software designs more understandable, flexible, and maintainable. Coined by Robert C. Martin (Uncle Bob), they help developers reduce complexity, avoid tight coupling, and create robust, scalable codebases.

- Single Responsibility Principle
- Open / Closed Principle
- Liskov Substitution Principle
- Interface Segmentation Principle
- Dependency Inversion Principle
